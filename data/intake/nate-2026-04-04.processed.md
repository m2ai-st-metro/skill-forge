# Nate's Newsletter Digest - 2026-04-04

**Subject:** I Tested Cowork, Lindy, Sauna, and Opal Against 3 Questions. The Best Scored 1 out of 4.

## TLDR

Nate tests four "outcome agents" (Cowork, Lindy, Sauna, Google Opal) against a 3-question framework: persistent memory, inspectable artifacts, and compounding context. Even the best (Cowork) scores roughly 1.25/3, exposing the core problem that knowledge work lacks the automated verification loops that make coding agents reliable. The thesis: memory-as-architecture you own, inspectable surfaces, and compounding context are the three durable principles -- tools will change, these won't.

## Extracted Ideas

### 1. Agent Evaluation Framework Skill

Build a `/eval-agent` skill that scores any agent tool against Nate's 3 structural questions (persistent memory, inspectable artifacts, compounding context) for a given use case, then generates a delegation spec that patches the weaknesses found.

- **Target platform:** Claude Code
- **Artifact type:** Skill
- **Complexity:** Weekend project
- **M2AI relevance:** HIGH -- directly applicable to evaluating tools M2AI builds and recommends to clients; could be packaged as a consulting deliverable.

### 2. Delegation Spec Generator

A skill/prompt that takes a tool name + task description, runs the 3-question evaluation, then outputs a pre-flight delegation spec: context packages for memory-weak tools, review gates for opacity-prone tools, compounding checkpoints for stateless tools. Nate calls this "writing the tests before the agent runs the work."

- **Target platform:** Claude Code
- **Artifact type:** Skill
- **Complexity:** Weekend project
- **M2AI relevance:** HIGH -- this is the "test suite for knowledge work" concept; directly useful for M2AI's own agent deployments and client handoffs.

### 3. Memory Architecture Auditor

An MCP server or skill that audits an agent system's memory layer against the 3-tier architecture Nate describes: key-value (preferences), relational (facts/IDs), vector (semantic similarity), event logs (history). Outputs a gap analysis and migration plan.

- **Target platform:** Claude Code
- **Artifact type:** Skill or MCP server
- **Complexity:** Weekend project
- **M2AI relevance:** HIGH -- M2AI already has Perceptor and Open Brain-style patterns; this formalizes the audit process and could be a paid consulting tool.

### 4. Context Compounding Monitor

A scheduled task that measures whether an agent system's context is actually compounding or rotting. Tracks: does run N+1 perform better than run N? Are memories being retrieved or just accumulated? Flags "memory rot" -- accumulated but unstructured context that degrades performance.

- **Target platform:** Claude Code
- **Artifact type:** Scheduled task + skill
- **Complexity:** Multi-sprint (needs metrics collection, baseline comparison, alerting)
- **M2AI relevance:** HIGH -- directly addresses a problem M2AI faces with its own agents (Life Engine, Starscream, Forge all have compounding loops that could rot silently).

### 5. Agent Inspectability Layer (Sheets-as-Memory Pattern)

Build an MCP server that exposes agent memory/state as a Google Sheet (or CSV/SQLite equivalent), inspired by Opal's accidentally-brilliant pattern. Makes any agent's memory human-inspectable and editable without specialized tooling.

- **Target platform:** Cross-platform (MCP server)
- **Artifact type:** MCP server
- **Complexity:** Weekend project
- **M2AI relevance:** MEDIUM -- interesting for client demos and transparency requirements, but M2AI already has SQLite-based inspection via Perceptor. The Google Sheets angle could be valuable for non-technical clients.

### 6. Recipe Caching System

Inspired by Sauna's "cached automations" concept: explain a workflow once, it runs automatically after that. Build a skill that lets users define agent recipes (prompt + tools + context requirements) that get stored and can be triggered by name or schedule. Essentially a skill-factory for recurring agent tasks.

- **Target platform:** Claude Code
- **Artifact type:** Skill (meta-skill / skill-factory)
- **Complexity:** Multi-sprint (needs recipe storage, parameterization, scheduling integration)
- **M2AI relevance:** MEDIUM -- Skill Forge already does some of this, but a lightweight "recipe" layer for non-developers would expand M2AI's addressable market.

### 7. Outcome Agent Comparison Dashboard

A web dashboard (or Obsidian page) that maintains a living comparison of outcome agents (Cowork, Lindy, Sauna, Opal, Obvious, etc.) scored against the 3-question framework, updated as products ship new features. Could be auto-maintained by a scheduled research agent.

- **Target platform:** Cross-platform (web or Obsidian)
- **Artifact type:** Scheduled task + agent
- **Complexity:** Multi-sprint
- **M2AI relevance:** MEDIUM -- good content marketing play for M2AI; positions Matthew as the evaluation authority. But requires ongoing maintenance.

### 8. Verification Gate Injector

A skill that takes any agent workflow/prompt and injects review gates at high-risk points -- specifically targeting the "silent failure" problem in knowledge work agents. Breaks black-box delegations into inspectable checkpoints. The "test suite for knowledge work" in practice.

- **Target platform:** Claude Code
- **Artifact type:** Skill
- **Complexity:** Weekend project
- **M2AI relevance:** HIGH -- directly solves a problem M2AI's clients face when deploying agents; could become a standard pre-deployment step in M2AI's delivery process.

### 9. Memory Portability Exporter

An MCP server or skill that exports agent memory from one platform's format into a portable, structured format (lifecycle-separated: preferences, project facts, session state, event logs). Addresses Nate's point that vendor lock-in on memory = vendor lock-in on compounding.

- **Target platform:** Cross-platform (MCP server)
- **Artifact type:** MCP server
- **Complexity:** Multi-sprint (needs adapters per platform)
- **M2AI relevance:** LOW -- conceptually sound but the market for memory portability is tiny right now. Worth tracking but not building yet.

### 10. Knowledge Work Test Suite Generator

The meta-idea from the newsletter: since knowledge work has no compiler/linter, build a skill that generates "test suites" for knowledge work outputs. Given a task spec (brief, memo, analysis), generate evaluation criteria, acceptance checks, and scoring rubrics that an agent can self-evaluate against before delivering.

- **Target platform:** Claude Code
- **Artifact type:** Skill
- **Complexity:** Weekend project
- **M2AI relevance:** HIGH -- this is the biggest gap in the entire agent ecosystem according to Nate's thesis. First-mover advantage if M2AI ships a credible version. Directly useful for every AiVA deployment.
