# Nate's Newsletter Digest - 2026-04-01

**Subject:** Every workaround you built for the last model is now breaking the next one. The 4-question audit + prompts to fix ...

## TLDR

Model step-changes (like Claude Mythos) don't just improve your AI systems -- they break the invisible compensating complexity you built around the last model's weaknesses. Nate presents a four-question audit framework plus four ready-made prompts to identify and strip scaffolding, procedural hacks, and org-chart dependencies before a new model makes them actively harmful.

## Buildable Ideas

### 1. Compensating Complexity Auditor (System Prompt Scanner)

Walk through a system prompt or pipeline description line by line, categorize every component as outcome logic, constraint, scaffolding, or duct tape, and suggest deletion tests for each.

- **Target platform:** Claude Code (primary), cross-platform (any LLM tool chain)
- **Artifact type:** Skill
- **Complexity:** Weekend project
- **M2AI relevance:** HIGH -- M2AI ships production prompts and agent systems constantly; a skill that audits system prompts for compensating complexity before model swaps is directly useful for every client engagement and internal project.

### 2. Outcome-Based Prompt Rewriter

Takes audit output (categorized prompt components) and produces a clean four-component prompt: outcome spec + constraints + tools + coordination pattern. Strips all procedural scaffolding.

- **Target platform:** Claude Code
- **Artifact type:** Skill
- **Complexity:** Weekend project
- **M2AI relevance:** HIGH -- pairs with the auditor above to form a two-step prompt modernization pipeline; every model upgrade cycle needs this.

### 3. Org-Level AI Dependency Mapper

Applies the compensating complexity diagnostic to team structure -- identifies which roles scale with model limitations, which survive any upgrade, and where Klarna-style risk exists (headcount/process hardcoded to current model weaknesses).

- **Target platform:** Cross-platform (Claude Code skill + standalone report generator)
- **Artifact type:** Skill / scheduled agent
- **Complexity:** Multi-sprint (requires org context ingestion, role taxonomy, nuanced analysis)
- **M2AI relevance:** MEDIUM -- valuable consulting deliverable for enterprise clients, but requires significant domain customization per client; not a generic drop-in tool.

### 4. Step-Change Readiness Plan Generator

Turns audit + dependency map outputs into a phased action plan sized to actual team capacity, starting with 30-minute quick wins. Produces a prioritized backlog of what to delete, test, and keep.

- **Target platform:** Claude Code
- **Artifact type:** Skill
- **Complexity:** Weekend project
- **M2AI relevance:** MEDIUM -- useful as a consulting accelerator but depends on the auditor and mapper outputs being solid first.

### 5. Model Upgrade Deletion Tester

Automated pipeline that takes a system prompt + test cases, systematically removes sections (procedural instructions, few-shot examples, format constraints), runs the test suite against a new model, and reports which deletions improve/maintain/degrade output quality.

- **Target platform:** Claude Code
- **Artifact type:** Skill (with CLI wrapper)
- **Complexity:** Multi-sprint (needs eval harness, model API integration, statistical comparison)
- **M2AI relevance:** HIGH -- this is the automation of Nate's core advice ("run your prompts with sections deleted, measure what gets worse"); would be a flagship M2AI tool for model migration consulting.

### 6. Verification Interception Rate Logger / Dashboard

Instrument existing verification steps, human review stages, and fallback paths to measure how often each actually changes output (not just how often it runs). Track over time to show when model upgrades make verification stages redundant.

- **Target platform:** Cross-platform (Python library + dashboard)
- **Artifact type:** MCP server or plugin
- **Complexity:** Multi-sprint (instrumentation hooks, storage, dashboard UI)
- **M2AI relevance:** MEDIUM -- operationally valuable for teams running multi-stage pipelines, but niche; better as a feature inside a larger observability product than standalone.

### 7. Planner-Worker-Judge Agent Harness Template

A reusable coordination pattern based on Cursor's validated architecture: planners decompose, workers execute independently, judge evaluates and iterates. No domain logic in the coordination layer. Ship as a Claude Code skill or Agent SDK template.

- **Target platform:** Claude Code / Agent SDK
- **Artifact type:** Agent template / skill
- **Complexity:** Weekend project (the pattern is well-defined; Cursor, Anthropic, and OpenAI have all published variants)
- **M2AI relevance:** HIGH -- M2AI already uses multi-agent patterns (yce-harness has orchestrator/worker/QA); a clean, reusable harness template accelerates every new agent build.

### 8. Agentic RAG Migration Skill

Converts pipeline RAG (chunking + re-ranking + hybrid search) to agentic RAG where the model controls retrieval decisions via tool use. Analyzes existing retrieval config, identifies stages that encode "retrieval intelligence" the model now has natively, and produces a tool-based replacement.

- **Target platform:** Claude Code
- **Artifact type:** Skill
- **Complexity:** Multi-sprint (needs to understand diverse RAG architectures, produce working tool definitions)
- **M2AI relevance:** MEDIUM -- relevant for clients with mature RAG pipelines, but the RAG landscape is fragmented; hard to make truly generic.

### 9. "What Stays" Classifier

Given a system prompt or pipeline config, classifies every component into: business rule (keeps), risk/compliance (keeps), model compensation (test for deletion), or domain hack (test for deletion). Distinct from the full auditor -- this is a quick triage pass.

- **Target platform:** Claude Code
- **Artifact type:** Skill
- **Complexity:** Weekend project
- **M2AI relevance:** HIGH -- fast, lightweight, immediately useful as a pre-flight check before any model swap; could be the entry point that upsells the full audit pipeline.

### 10. Bitter Lesson Scorecard for Agent Architectures

Scores an agent system design against the Bitter Lesson principle: how much "how" is encoded vs. "what". Flags procedural lock-in, hardcoded orchestration, and domain hacks that bet against model improvement. Produces a simplification roadmap.

- **Target platform:** Claude Code
- **Artifact type:** Skill
- **Complexity:** Weekend project
- **M2AI relevance:** HIGH -- unique positioning; no one else is packaging the Bitter Lesson as a practical audit tool for agent builders. Strong content/consulting angle.
